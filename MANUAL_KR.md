## 개요

NAI Auto Generator V4.5는 NovelAI의 이미지 생성 API를 활용한 강력한 데스크톱 애플리케이션입니다. 
NovelAI 웹 인터페이스에서 제공하지 않는 고급 기능들을 제공하여, 더욱 효율적이고 정교한 AI 이미지 생성을 가능하게 합니다.

### 주요 특징

- **V4 모델 완전 지원**: NovelAI의 최신 모델 NovelAI Diffusion V4.5 Full 지원
- **고급 캐릭터 제어**: Character Prompts와 Character Reference를 통한 정밀한 캐릭터 생성
- **이미지 편집 기능**: img2img, Enhancement, Inpainting을 통한 이미지 수정 및 향상
- **자동화 기능**: 와일드카드, 배치 생성, 자동 생성 등 워크플로우 최적화
- **다국어 지원**: 한국어, 영어, 일본어, 중국어 인터페이스 제공

## 기본 기능

NAI Auto Generator를 사용하기 위한 기본적인 기능들을 안내합니다.

### 로그인

NovelAI 계정으로 로그인하여 이미지 생성 기능을 활성화합니다.

#### 로그인 방법 (ID / 비밀번호)

1. **메뉴에서 로그인**
   - `Files` → `Log in` 선택

2. **"ID / Password" 탭 선택**
   - Username: NovelAI 계정 이메일 또는 사용자명
   - Password: NovelAI 계정 비밀번호
   - Google ID 연동으로 ID 생성시, Web에서 User Settings → Account 에서 Password 변경(설정) 후 사용

3. **자동 로그인** (선택사항)
   - `Auto login next time` 체크박스 활성화
   - 다음 실행 시 자동으로 로그인됨

4. **로그인 확인**
   - 상단 상태바에 "로그인 완료" 메시지 표시
   - ANLAS 잔액 표시

#### 로그인 방법 (API 키)

NovelAI Persistent API Token을 사용하여 로그인할 수 있습니다.

1. **토큰 발급**
   - NovelAI 사이트 → Settings → Account → "Get Persistent API Token" 클릭
   - `pst-...`로 시작하는 토큰을 복사

2. **"API 키" 탭 선택**
   - 로그인 대화상자에서 "API 키" 탭 선택
   - 발급받은 토큰 입력 (`pst-`로 시작)
   - `Show` / `Hide` 버튼으로 토큰 표시 여부 전환

3. **로그인 버튼 클릭**

#### 주의사항

- 사용자 이름과 비밀번호는 NovelAI 서버로만 전송됩니다
- 본 앱의 서버로는 전송되지 않습니다
- 자동 로그인 활성화 시 로컬에 암호화되어 저장됩니다

### 프롬프트 입력

이미지 생성을 위한 프롬프트와 네거티브 프롬프트를 입력합니다.

프롬프트 입력 영역은 **탭(Tab) UI** 형태로 구성됩니다.
- **Prompt 탭**: 생성 프롬프트 입력
- **Negative Prompt 탭**: 네거티브 프롬프트 입력
- 각 탭 제목에 현재 입력된 **문자 수**가 실시간으로 표시됩니다 (예: `Prompt (248)`)

#### Prompt (프롬프트)

생성하고 싶은 이미지의 내용을 설명합니다.

**기본 작성 방법**:
```
1girl, blue eyes, long hair, standing, outdoor, smile
```

**구조 권장사항**:
```
[주제], [외형 특징], [포즈/동작], [배경], [분위기/감정], [품질 태그]
```

**예시**:
```
1girl, blue dress, long blonde hair, standing in garden, 
happy smile, sunlight, masterpiece, best quality, high quality
```

#### Negative Prompt (네거티브 프롬프트)

생성 시 제외하고 싶은 요소를 명시합니다.

**일반적인 네거티브 프롬프트**:
```
lowres, bad quality, worst quality, bad anatomy, 
missing fingers, extra digits, blurry, text, watermark
```

#### 프롬프트 고급 문법

##### 1. 가중치 (Emphasis)

**중괄호 사용**:
- `{태그}`: 1.05배 강조
- `{{태그}}`: 1.1025배 강조 (중첩 가능)
- `{{{태그}}}`: 1.157625배 강조

**대괄호 사용**:
- `[태그]`: 0.95배 약화
- `[[태그]]`: 0.9025배 약화 (중첩 가능)

**정확한 값 지정** (Colon Emphasis):
```
1.2::masterpiece ::
0.8::simple background ::
-2::multiple views ::
```
- 형식: `가중치::텍스트 ::`
- 가중치 범위: 일반적으로 0.5 ~ 1.5
- 마이너스 가중치: 배제하고 싶은 요소가 있을 경우 사용 (부정 프롬프트에는 권장하지 않음)

##### 2. Artist 태그

특정 아티스트의 스타일을 적용합니다.
```
{artist: artist_name}
```

예시:
```
1girl, {artist:abcd}, fantasy art style
```

##### 3. 년도 태그

특정 년도의 스타일을 참조합니다.
```
year 2024, year 2025
```

##### 4. 품질 태그 조합

```
masterpiece, best quality, high quality, amazing quality, 
very aesthetic, intricate details, detailed shading
```

#### 구문 강조 (Syntax Highlighting)

프롬프트 입력 시 자동으로 문법이 강조 표시됩니다:
- **중괄호 강조**: 주황색으로 표시
- **대괄호 약화**: 파란색으로 표시
- **Colon Emphasis**: 숫자와 구분자가 다른 색으로 표시
- **Artist 태그**: 고유한 색상으로 강조

### 이미지 생성

기본적인 이미지 생성 방법을 안내합니다.

#### Options & Settings

"Image Options"와 "Advanced Settings"가 하나의 **Options & Settings** 패널로 통합되어 있습니다.  
좌측 패널 상단에 위치하며, 모든 생성 파라미터를 한 곳에서 설정할 수 있습니다.

##### Model (모델)

NovelAI 생성 모델을 선택합니다.

| 표시 이름 | 설명 |
|----------|------|
| **NAI Diffusion 4.5 Full** | 최신 V4.5 풀 모델 (기본값, 권장) |
| **NAI Diffusion 4.5 Curated** | V4.5 Curated 모델 |
| **NAI Diffusion 4 Full** | V4 풀 모델 |
| **NAI Diffusion 4 Curated** | V4 Curated 모델 |
| **NAI Diffusion 3** | 구버전 V3 모델 |

**참고**:
- V3 모델 선택 시 Character Prompts, Character Reference 등 V4 전용 기능은 비활성화됩니다

##### Size (해상도)

드롭다운 메뉴에서 원하는 해상도 선택:

**Normal 해상도**:
- 1024×1024 (정사각형)
- 832×1216 (세로)
- 1216×832 (가로)

**Large 해상도**:
- 1472×1472 (정사각형)
- 1024×1536 (세로)
- 1536×1024 (가로)

**Wallpaper 해상도**:
- 1088×1920 (세로 와이드)
- 1920×1088 (가로 와이드)

**Small 해상도**:
- 640×640, 512×768, 768×512

**참고**:
- Opus 티어 기준, Normal 해상도는 ANLAS 소모하지 않음
- Large 및 Wallpaper 해상도는 ANLAS 소비 (Options에서 비활성화 가능)
- Image Enhancement는 Normal 해상도만 지원

##### Width / Height (너비 / 높이)

해상도를 직접 수치로 입력할 수 있습니다.
- 범위: 64 ~ 2048 픽셀
- 64의 배수로 입력 권장

##### Random (무작위 해상도)

체크박스 활성화 시 매 생성마다 무작위 해상도 선택됩니다.

##### Sampler (샘플러)

이미지 생성 알고리즘을 선택합니다.

**주요 샘플러**:
- **k_euler_ancestral**: 기본값, 다양한 결과
- **k_euler**: 일관된 결과
- **ddim**: 세밀한 제어
- **k_dpmpp_2s_ancestral**: 고품질, 시간 다소 소요
- **k_dpmpp_sde**: 매우 고품질, 시간 많이 소요

##### Noise Schedule (노이즈 스케줄)

노이즈 적용 방식을 선택합니다.
- **기본값**: karras
- **옵션**: native, karras, exponential, polyexponential

##### Steps (생성 단계)

생성 과정의 반복 횟수입니다.
- 기본값: 28
- 범위: 1~50
- 높을수록: 더 정밀하지만 시간 증가
- 권장: 28 (기본) ~ 40 (고품질)

##### Prompt Guidance (프롬프트 가이던스)

프롬프트에 대한 AI의 충실도를 조절합니다.
- 기본값: 5.0
- 낮은 값: 더 창의적이고 예측 불가능한 결과
- 높은 값: 프롬프트에 더 충실한 결과

##### Seed (시드)

생성 결과를 재현 가능하게 하는 난수 값입니다.
- **Random**: 매번 다른 시드 사용 (다양한 결과)
- **Fix**: 고정된 시드 사용 (동일한 결과 재현)
- 범위: 0 ~ 4,294,967,295

**활용**:
- 마음에 드는 결과의 시드를 고정하여 비슷한 이미지 생성
- 프롬프트만 약간 수정하여 변형 생성

##### Prompt Guidance Rescale

CFG Rescale 값을 설정합니다.
- 기본값: 0 (비활성)
- 범위: 0.0 ~ 1.0

##### Variety+ (다양성 증가)

생성 초기 단계에서 CFG를 건너뛰어 더 다양한 결과를 만들어내는 기능입니다.
- 체크박스로 활성화/비활성화
- 같은 시드로도 더 다양한 변형 생성 가능

##### Legacy Mode (레거시 모드)

V3 스타일의 프롬프트 처리 방식을 사용합니다.
- 일반적인 사용에서는 비활성화 권장

#### 이미지 생성 버튼

생성 버튼은 우측 결과 이미지 영역 **하단 바**에 위치합니다.

| 버튼 | 기능 |
|------|------|
| **1회 생성** | 현재 설정으로 이미지 1장 생성 |
| **연속 생성** | 연속 생성 옵션 대화상자를 열어 여러 장 자동 생성 |
| **이미지 저장** | 현재 결과 이미지를 파일로 저장 |
| **크기 초기화** | 결과 이미지 표시 창을 기본 크기로 복원 |

#### 결과 프롬프트 표시 (오버레이)

결과 이미지 하단 바 우측의 **"프롬프트 결과 표시"** 체크박스를 사용합니다.

- **활성화**: 결과 프롬프트가 이미지 하단에 반투명 오버레이로 표시
- **비활성화**: 오버레이 숨김
- 오버레이 텍스트 색상은 `Files` → `Option` → 프롬프트 설정에서 변경 가능

### 설정 저장 및 불러오기

현재 프롬프트와 파라미터 설정을 저장하고 불러올 수 있습니다.

#### 설정 저장

1. **메뉴 사용**
   - `Files` → `Save Settings` (단축키: Ctrl+S)
   
2. **파일명 입력**
   - 원하는 이름으로 .txt 파일 저장
   
3. **저장 내용**
   - 프롬프트 (Prompt)
   - 네거티브 프롬프트 (Negative Prompt)
   - 캐릭터 프롬프트 (Character Prompts)
   - 모든 생성 파라미터 (해상도, 샘플러, 스텝 등)
   - Character Reference 설정
   - Image to Image 설정

#### 설정 불러오기

1. **메뉴 사용**
   - `Files` → `Load Settings` (단축키: Ctrl+O)
   
2. **파일 선택**
   - 저장된 .txt 설정 파일 선택
   
3. **적용 확인**
   - 프롬프트 및 모든 파라미터가 복원됨

#### 이미지에서 설정 불러오기

NovelAI로 생성한 PNG 이미지는 메타데이터를 포함하고 있어 설정을 복원할 수 있습니다.

1. **드래그 앤 드롭**
   - PNG 또는 WEBP 이미지를 메인 화면에 드롭
   
2. **메뉴 사용**
   - `Files` → `Open file`
   - PNG, WEBP 이미지 선택

3. **복원 내용**
   - 프롬프트 및 모든 생성 파라미터
   - 캐릭터 프롬프트 (있는 경우)
   - 결과 이미지 미리보기

### Options 설정

애플리케이션의 전역 설정을 관리합니다.

#### Options 대화상자 열기

- `Files` → `Option` 메뉴 선택

#### 폴더 경로 설정

각 기능별 기본 폴더 위치를 지정합니다.

##### 결과 저장 폴더

생성된 이미지가 저장되는 위치입니다.
- **기본 경로**: `사용자폴더/NAI-Auto-Generator/results`
- **변경 방법**: 찾아보기 버튼으로 원하는 폴더 선택
- **용도**: 모든 생성 이미지가 이 폴더에 저장됨

##### 세팅 파일 저장 폴더

설정 파일(.txt)이 저장되는 위치입니다.
- **기본 경로**: `사용자폴더/NAI-Auto-Generator/settings`
- **용도**: Save Settings로 저장한 파일 위치

##### 와일드카드 파일 폴더

와일드카드 텍스트 파일들이 위치하는 폴더입니다.
- **기본 경로**: `사용자폴더/NAI-Auto-Generator/wildcards`
- **용도**: `##파일명##` 형식으로 사용되는 와일드카드 파일들

##### 태그 완성 파일 경로

태그 자동완성에 사용되는 CSV 파일 경로입니다.
- **기본 파일**: `tag_completion.csv`
- **용도**: 프롬프트 입력 시 자동완성 제안
- **변경 시**: 태그 새로고침 권장

#### 로그 설정

문제 해결을 위한 로그 기록 설정입니다.

##### 디버그 모드 활성화

체크박스로 디버그 모드를 켜거나 끌 수 있습니다.
- **비활성화** (기본값): 기본적인 정보만 기록
- **활성화**: 상세한 디버그 정보 기록
  - 함수 호출 추적
  - 변수 값 기록
  - 상세한 에러 정보

**사용 시기**:
- 문제가 발생했을 때
- 개발자에게 버그 리포트 시

##### 로그 상세 수준

로그에 기록되는 정보의 양을 조절합니다.
- **NORMAL**: 일반적인 작업 정보만 기록
- **DETAILED**: 더 상세한 정보 기록

##### 로그 파일 위치

로그 파일이 저장되는 폴더를 지정합니다.
- **기본 경로**: `사용자폴더/NAI-Auto-Generator/logs`
- **파일명 형식**: `nai_generator_YYYYMMDD.log`
- **로그 폴더 열기 버튼**: 로그 파일 위치를 탐색기로 열기

#### 연속생성 설정

Quick Generation 버튼의 매수를 커스터마이징합니다.

##### Quick Generation 매수

자동 생성 대화상자의 4개 프리셋 버튼 매수를 설정합니다.
- **1번 버튼**: 기본값 5장 (변경 가능: 1~9999)
- **2번 버튼**: 기본값 10장
- **3번 버튼**: 기본값 50장
- **4번 버튼**: 기본값 100장

**활용**:
- 자주 사용하는 매수로 커스터마이징
- 예: 5, 20, 100, 500으로 설정

#### 프롬프트 설정

프롬프트 입력 관련 기능을 설정합니다.

##### 구문 강조

프롬프트 입력 시 문법을 색상으로 강조 표시합니다.
- 중괄호, 대괄호, Colon Emphasis 등을 시각적으로 구분
- 프롬프트 작성 시 문법 오류 방지

##### 오버레이 텍스트 색상

결과 이미지 위에 표시되는 프롬프트 오버레이의 텍스트 색상을 변경합니다.
- 색상 버튼 클릭 → 색상 선택 대화상자에서 원하는 색상 선택
- 기본값: 흰색 (#ffffff)
- 설정 즉시 적용 및 재시작 후에도 유지

#### 글꼴 설정

##### 글꼴 크기

프롬프트 입력창의 글꼴 크기를 조절합니다.
- 범위: 8 ~ 32pt
- 기본값: 18pt
- 가독성에 맞게 조절 가능

#### 태그 설정

##### 태그 새로고침

태그 자동완성 목록을 다시 로드합니다.
- **사용 시기**: 
  - 태그 CSV 파일을 수정한 후
  - 새로운 태그 파일을 추가한 후
- **방법**: 
  - Options에서 "태그 새로고침" 버튼 클릭
  - 또는 `Files` → `Reload Tag Completion` 메뉴

**참고**: 애플리케이션 재시작 없이 태그 목록 업데이트 가능

#### 테마 설정

애플리케이션 UI 테마를 변경합니다.
- **기본 테마**: 시스템 설정 따름
- **밝은 테마**: 밝은 배경 사용
- **어두운 테마**: 어두운 배경 사용 (다크 모드)

**적용**:
- 테마 선택 후 즉시 UI에 반영됨
- 재시작 불필요

#### 파일명 포맷 설정

생성되는 이미지의 파일명 형식을 커스터마이징합니다.

##### 지원 플레이스홀더

- `[datetime]`: 날짜와 시간 (예: 20250118_143052)
- `[date]`: 날짜만 (예: 20250118)
- `[time]`: 시간만 (예: 143052)
- `[prompt]`: 프롬프트 텍스트
- `[character]`: 캐릭터 프롬프트
- `[seed]`: 시드 값

##### 단어 수 제한

- 파일명이 너무 길어지는 것 방지

##### 포맷 예시

- `[datetime]_[prompt]` (기본값)
- `[date]_[seed]_[prompt]`
- `[character]_[datetime]`
- `[prompt]_[time]_s[seed]`

**장점**:
- 파일 정리 및 검색 용이
- 과도하게 긴 파일명 오류 방지
- 원하는 방식으로 파일 관리 가능

## 주요 기능

### Character Prompts (V4)

V4 모델에서 도입된 기능으로, 이미지 내에 여러 캐릭터를 개별적으로 지정하고 위치를 제어할 수 있습니다.

#### 기본 사용법

1. **메뉴에서 활성화**
   - `View` → `Character Prompts` 선택 (또는 `F1` 단축키)
   
2. **캐릭터 추가**
   - `+ 캐릭터 추가` 버튼 클릭
   - 각 캐릭터마다 별도의 프롬프트 입력 가능
   
3. **위치 지정 방식 선택**
   - **AI Position Selection (AI 위치 선택)**: AI가 자동으로 최적의 위치 결정
   - **Manual Position (수동 위치)**: 좌표(X, Y)를 직접 입력하여 정확한 위치 지정
     - X, Y 값은 0.0 ~ 1.0 사이의 값 (0.0 = 왼쪽/위, 1.0 = 오른쪽/아래)
     - 예: (0.3, 0.5) = 왼쪽에서 30%, 위에서 50% 위치

4. **네거티브 프롬프트 추가** (선택사항)
   - 각 캐릭터에 개별 네거티브 프롬프트 지정 가능
   - `네거티브 프롬프트 추가` 체크박스로 활성화

#### 활용 팁

- **다중 캐릭터 구도**: 2~3명의 캐릭터를 배치할 때 AI 위치 선택을 사용하면 자연스러운 구도 생성
- **정밀 배치**: 특정 위치에 캐릭터를 배치하고 싶다면 수동 위치 모드 사용
- **캐릭터별 스타일 조절**: 각 캐릭터의 프롬프트에 서로 다른 아티스트 태그나 스타일 적용 가능

#### 주의사항

- Character Prompts는 Character Reference와 함께 사용할 수 있습니다
- Vibe Transfer와는 호환되지 않습니다 (Character Reference와 Vibe Transfer는 동시 사용 불가)

### Character Reference

특정 캐릭터의 외형을 일관되게 유지하면서 다양한 포즈와 구도로 생성할 수 있는 기능입니다.

#### 기본 사용법

1. **메뉴에서 활성화**
   - `View` → `Character Reference` 선택 (또는 `F1` 단축키)

2. **참조 이미지 선택**
   - `Select Image` 버튼 클릭하여 참조할 캐릭터 이미지 선택
   - 또는 이미지를 드래그 앤 드롭

3. **파라미터 조절**
   - **Fidelity (충실도)**: 0.0 ~ 1.0 (기본값: 1.0)
     - 높을수록 참조 이미지와 유사하게 생성
     - 낮을수록 더 자유로운 변형 허용
   
   - **Style Aware (스타일 인식)**: 체크박스
     - 활성화: 캐릭터의 외형과 스타일을 모두 참조
     - 비활성화: 캐릭터의 외형만 참조 (스타일은 프롬프트에 따름)

4. **프롬프트 입력 및 생성**
   - 원하는 포즈, 배경, 상황을 프롬프트에 입력
   - 캐릭터의 외형은 참조 이미지를 기반으로 유지됨

#### 활용 팁

- **캐릭터 일관성**: 시리즈물이나 여러 장면에서 동일한 캐릭터 유지
- **다양한 포즈**: 참조 이미지의 포즈와 다른 포즈도 자유롭게 생성 가능
- **Fidelity 조절**: 
  - 0.8~1.0: 매우 유사한 캐릭터 생성 (얼굴 특징 강하게 유지)
  - 0.5~0.7: 캐릭터 특징을 유지하면서 변형 허용
  - 0.3~0.5: 부분적 특징만 참조 (더 자유로운 생성)

#### 권장 이미지 사양

- **해상도**: 1024×1536 (세로), 1472×1472 (정사각형), 1536×1024 (가로)
- **포맷**: PNG, JPG, WEBP
- 앱이 자동으로 권장 해상도로 조정하므로 원본 크기는 자유롭게 선택 가능

#### 비용

- Character Reference 사용 시 생성당 **5 ANLAS** 추가 차감

### Image to Image (img2img)

기존 이미지를 기반으로 새로운 이미지를 생성하거나 수정하는 기능입니다.

#### 기본 사용법

1. **메뉴에서 활성화**
   - `View` → `Image to Image (img2img)` 선택 (또는 `F2` 단축키)

2. **원본 이미지 선택**
   - `Select Image` 버튼 클릭
   - 또는 이미지를 위젯에 드래그 앤 드롭
   - 선택된 이미지는 164×198px 썸네일로 미리보기 표시

3. **파라미터 조절**
   - **Strength (강도)**: 0.0 ~ 1.0 (기본값: 0.7)
     - 높을수록 원본 이미지에서 더 많이 변형
     - 낮을수록 원본 이미지와 유사하게 유지
   
   - **Noise (노이즈)**: 0.0 ~ 1.0 (기본값: 0.0)
     - 생성 과정에 추가되는 무작위성
     - 높을수록 더 다양한 결과 생성

4. **프롬프트 입력 및 생성**
   - 원하는 변경사항을 프롬프트에 입력
   - 원본 이미지의 구도와 구조는 어느 정도 유지됨

#### 활용 사례

- **스타일 변경**: 사진을 애니메이션 스타일로 변환
- **구도 유지**: 스케치나 러프를 완성도 높은 이미지로 변환
- **부분 수정**: 이미지의 일부 요소만 변경
- **업스케일 준비**: Enhancement 전에 이미지 정제

#### Strength 값 가이드

- **0.3 ~ 0.5**: 원본과 매우 유사, 색상이나 디테일만 변경
- **0.5 ~ 0.7**: 균형잡힌 변형, 구도는 유지하면서 스타일 변경 가능
- **0.7 ~ 0.9**: 원본을 참고만 하고 크게 변형, 새로운 요소 추가 가능

#### 주의사항

- img2img 활성화 시 autoSmea가 자동으로 비활성화됩니다 (API 요구사항)
- 원본 이미지의 해상도가 생성 해상도와 다를 경우 자동으로 조정됩니다

### Image Enhancement

생성된 이미지를 업스케일하고 디테일을 향상시키는 기능입니다.

#### 기본 사용법

1. **메뉴에서 활성화**
   - `View` → `Image Enhance` 선택 (또는 `F3` 단축키)

2. **이미지 선택**
   - **Use Current Image**: 방금 생성한 이미지를 자동으로 사용
   - **Select Image**: 파일에서 이미지 선택
   - **Select Folder**: 여러 이미지를 한 번에 처리 (배치 Enhancement)

3. **업스케일 비율 선택**
   - **1x**: 원본 크기 유지 (디테일만 향상)
   - **1.5x**: 해상도 1.5배 증가

4. **파라미터 조절**
   - **Strength (강도)**: 0.01 ~ 0.99 (기본값: 0.40)
     - 디테일 향상 강도 조절
     - 너무 높으면 원본과 달라질 수 있음
   
   - **Noise (노이즈)**: 0.00 ~ 0.99 (기본값: 0.00)
     - 생성 과정의 무작위성
     - 일반적으로 0.00 유지 권장

5. **생성 실행**
   - `Enhance 시작` 버튼 클릭
   - 배치 모드의 경우 자동으로 순차 처리

#### 해상도 매핑 (1.5x)

| 원본 해상도 | Enhancement 해상도 |
|------------|-------------------|
| 1024×1024 | 1536×1536 |
| 832×1216 | 1280×1856 |
| 1216×832 | 1856×1280 |

#### 배치 Enhancement

1. `Select Folder` 버튼으로 이미지 폴더 선택
2. NovelAI 메타데이터가 포함된 이미지 자동 감지
3. `Enhance 시작` 버튼으로 일괄 처리 시작
4. 각 이미지 사이 3초 대기 (API 제한 방지)
5. 진행 상황 실시간 표시

#### 활용 팁

- **고품질 출력**: Strength 0.3~0.5로 안전하게 디테일 향상
- **배치 처리**: 여러 이미지를 한 번에 업스케일할 때 유용
- **메타데이터 필수**: Enhancement는 원본 이미지의 NovelAI 메타데이터 필요

#### 지원 해상도

- **Normal 해상도만 지원**: 1024×1024, 832×1216, 1216×832
- **Large 해상도 미지원**: 1472×1472, 1024×1536, 1536×1024
- Large 해상도 이미지 선택 시 경고 메시지 표시

#### 비용

- Enhancement 사용 시 생성당 **추가 ANLAS** 차감 (업스케일 비율에 따라 다름)

### Image Inpainting

이미지의 특정 영역만 선택적으로 재생성하는 기능입니다.

#### 기본 사용법

1. **Image to Image로 이미지 로드**
   - img2img 위젯에서 수정할 이미지 선택

2. **마스크 페인팅 대화상자 열기**
   - `Paint Mask` 버튼 클릭

3. **마스크 영역 지정**
   - 마우스로 드래그하여 재생성할 영역 칠하기
   - 칠해진 영역은 빨간색 오버레이로 표시
   - 8×8 픽셀 그리드 단위로 정밀 작업 가능

4. **브러시 크기 조절**
   - 슬라이더로 1~20 그리드 셀 조절 (기본값: 3)
   - 크기 1 = 8×8 픽셀
   - 크기 3 = 24×24 픽셀 (기본)
   - 크기 20 = 160×160 픽셀 (최대)

5. **마스크 반전** (선택사항)
   - `Invert Mask` 체크박스 활성화
   - 칠하지 않은 영역을 재생성

6. **확인 및 생성**
   - `OK` 버튼 클릭
   - 메인 화면에서 프롬프트 입력 후 생성
   - 마스크 영역만 재생성됨

#### 활용 사례

- **부분 수정**: 얼굴, 손, 배경 등 특정 부분만 다시 생성
- **요소 제거**: 원치 않는 요소를 마스킹하고 재생성
- **요소 추가**: 빈 영역을 마스킹하고 새로운 요소 추가
- **스타일 통일**: 이질적인 부분을 나머지와 어울리게 수정

#### 마스크 반전 활용

- **일반 모드**: 칠한 영역 재생성 (일반적 사용)
- **반전 모드**: 칠하지 않은 영역 재생성 (대부분을 수정할 때 유용)

#### 주의사항

- 마스크는 대화상자를 닫아도 유지됩니다 (다시 열면 이전 마스크 표시)
- 8×8 그리드 단위로 작동하므로 매우 정밀한 작업 가능
- 녹색 그리드 선으로 블록 경계 확인 가능

### Vibe Transfer

참조 이미지의 분위기와 스타일을 새 이미지에 적용하는 기능입니다.

#### 기본 사용법

1. **참조 이미지 선택**
   - Reference Image 위젯에서 `Select Image` 클릭
   - 또는 이미지를 드래그 앤 드롭

2. **파라미터 조절**
   - **Reference Strength (참조 강도)**: 0.0 ~ 1.0
     - 참조 이미지의 영향력 조절
     - 높을수록 참조 이미지와 유사한 스타일 생성

3. **프롬프트 입력 및 생성**
   - 원하는 내용을 프롬프트에 입력
   - 참조 이미지의 색감, 분위기, 스타일이 적용됨

#### Character Reference와의 차이점

| 기능 | Character Reference | Vibe Transfer |
|-----|-------------------|--------------|
| 목적 | 캐릭터 외형 유지 | 분위기/스타일 전달 |
| 참조 대상 | 특정 캐릭터 | 전체적인 느낌 |
| 활용 | 동일 캐릭터 다른 포즈 | 유사한 분위기 이미지 |
| 호환성 | Character Prompts와 사용 가능 | Character Reference와 동시 사용 불가 |

#### 활용 팁

- **스타일 통일**: 시리즈 이미지의 일관된 느낌 유지
- **색감 조절**: 특정 색감이나 조명 분위기 전달
- **참조 강도**: 0.3~0.6이 일반적으로 자연스러운 결과

#### 주의사항

- Character Reference와 Vibe Transfer는 동시에 사용할 수 없습니다
- 두 기능 중 하나만 선택하여 사용하세요

## 고급 설정

> 아래 설정들은 모두 좌측 패널의 **Options & Settings** 섹션에서 찾을 수 있습니다.

### Variety+ (다양성 증가)

생성 초기 단계에서 CFG를 건너뛰어 더 다양한 결과를 만들어내는 기능입니다.

- **위치**: Options & Settings 섹션 하단
- **체크박스**: Variety+ 활성화/비활성화
- **효과**: 같은 시드로도 더 다양한 변형 생성 가능
- **API 파라미터**: `skip_cfg_above_sigma = 19` 설정

### Noise Schedule (노이즈 스케줄)

노이즈 적용 방식을 조절하는 고급 설정입니다.

- **기본값**: karras
- **옵션**: native, karras, exponential, polyexponential
- **효과**: 이미지 생성 과정의 노이즈 분포 조절

### Legacy Mode (레거시 모드)

V3 모델의 프롬프트 처리 방식을 사용하는 모드입니다.

- **용도**: V3 스타일의 결과가 필요한 경우
- **일반 사용**: 비활성화 권장

### Sampler (샘플러)

이미지 생성 알고리즘을 선택합니다.

- **k_euler_ancestral**: 기본 샘플러, 다양한 결과
- **k_euler**: 일관된 결과
- **ddim**: 세밀한 제어 가능
- 기타 여러 샘플러 지원

### Steps (스텝 수)

생성 과정의 반복 횟수입니다.

- **기본값**: 28
- **범위**: 1~50
- **효과**: 높을수록 더 정밀하지만 시간이 오래 걸림

### Prompt Guidance (프롬프트 가이던스)

프롬프트에 대한 AI의 충실도를 조절합니다.

- **기본값**: 5.0
- **범위**: 0.0~30.0
- **낮은 값**: 더 창의적이고 예측 불가능한 결과
- **높은 값**: 프롬프트에 더 충실한 결과

## 와일드카드 시스템

와일드카드는 프롬프트에서 무작위로 선택되는 텍스트 패턴을 정의하는 기능입니다.

### 기본 문법

#### 단순 와일드카드
```
##파일명##
```
- `wildcards` 폴더의 `파일명.txt` 파일에서 무작위로 한 줄 선택

#### 반복 와일드카드 (Loopcard)
```
##파일명*N##
```
- N번 동안 같은 값 유지
- 예: `##characters*3##` - 3번 생성 동안 같은 캐릭터 유지

#### 중첩 선택 (Lessthan)
```
{옵션1|옵션2|옵션3}
```
- 여러 옵션 중 하나를 무작위로 선택
- 예: `{red|blue|green} dress` → "red dress" 또는 "blue dress" 또는 "green dress"

### 와일드카드 파일 작성

1. **파일 위치**
   - `wildcards` 폴더에 `.txt` 파일 생성
   - 메뉴: `Folders` → `Wildcards Folder` (F6)로 폴더 열기

2. **파일 내용**
   - 한 줄에 하나의 옵션
   - 빈 줄은 무시됨
   
   예시 (`characters.txt`):
   ```
   1girl, blue eyes, long hair
   1boy, red hair, short hair
   1girl, green eyes, ponytail
   ```

3. **프롬프트에 사용**
   ```
   ##characters##, standing, outdoors
   ```
   → 매번 다른 캐릭터로 생성

### 고급 활용

#### Loopcard로 연속성 유지
```
##pose*5##, ##expression*5##, ##background##
```
- pose와 expression은 5번 동안 유지
- background는 매번 변경

#### 조건부 선택
```
{##summer_clothes##|##winter_clothes##}
```
- 여름옷 또는 겨울옷 와일드카드 중 선택

#### 가중치와 함께 사용
```
{1.2::##strong_features##::0.8|##soft_features##}
```

## 자동 생성 기능

여러 장의 이미지를 자동으로 연속 생성하는 기능입니다.

### 자동 생성 대화상자

1. **열기**
   - 결과 이미지 하단 바의 **"연속 생성"** 버튼 클릭

2. **Quick Generation (빠른 생성)**
   - 5장, 10장, 50장, 100장 버튼으로 즉시 시작
   - Options에서 각 버튼의 매수 커스터마이징 가능

3. **Manual Input (수동 입력)**
   - 생성 횟수: 원하는 매수 입력 (-1 = 무제한)
   - 생성 간격: 각 생성 사이 대기 시간 (초)
   - 오류 무시: 체크 시 생성 오류 발생해도 계속 진행

4. **시작 및 중지**
   - 연속 생성 시작 후 버튼이 **"생성 중지"**로 변경됨
   - "생성 중지" 버튼 클릭으로 중지

### 배치 설정 파일

여러 설정을 순차적으로 적용하며 생성할 수 있습니다.

1. **설정 파일 준비**
   - 여러 개의 `.txt` 설정 파일 준비
   - 각 파일에 다른 프롬프트나 파라미터 저장

2. **배치 생성 실행**
   - 파일 선택 시 여러 개 선택
   - 자동 생성 시작
   - 각 설정으로 순차적으로 생성

### 상태 표시

자동 생성 중 상태바에 진행 상황 표시:
- "연속 생성 중 (5/10)"
- "다음 생성 대기 중... (3초)"
- "에러 발생. 5초 후 재시도..."

## 단축키 및 팁

### 주요 단축키

#### View 메뉴
- `F1`: Character Reference 토글
- `F2`: Image to Image 토글
- `F3`: Image Enhance 토글
- `F11`: Results Panel 토글
- `Ctrl+R`: 레이아웃 초기화
- `Ctrl+Shift+I`: 이미지 크기 초기화

#### Folders 메뉴
- `F5`: Results Folder 열기
- `F6`: Wildcards Folder 열기
- `F7`: Settings Folder 열기

#### File 메뉴
- `Ctrl+S`: 설정 저장
- `Ctrl+O`: 설정 불러오기

### 프롬프트 작성 팁

#### 효과적인 프롬프트 구조
```
[주제], [세부 특징], [스타일], [품질 태그]
```

예시:
```
1girl, blue eyes, long hair, standing, outdoor, 
{artist: xxx}, official art, masterpiece, best quality
```

#### 가중치 사용
- `{태그}`: 1.05배 강조
- `{{태그}}`: 1.1025배 강조 (중첩 가능)
- `[태그]`: 0.95배 약화
- `1.2::태그::`: 1.2배 강조 (정확한 값 지정)

#### Artist 태그
```
{artist: artist_name}
```
- NovelAI가 학습한 특정 아티스트 스타일 적용

### 효율적인 워크플로우

#### 1. 기본 이미지 생성
- Character Prompts로 원하는 구도 생성
- 여러 번 생성하여 마음에 드는 구도 선택
- Seed를 고정하여 일관된 결과 유지

#### 2. 세부 조정
- Character Reference로 캐릭터 일관성 확보
- Image to Image로 부분적 수정
- Inpainting으로 문제 영역만 재생성

#### 3. 최종 마무리
- Enhancement로 해상도 향상 및 디테일 강화

#### 4. 대량 생성
- 와일드카드로 다양한 변형 자동 생성
- 자동 생성 기능으로 여러 장 연속 생성

### 문제 해결

#### 생성 실패 시
1. 프롬프트 길이 확인 (너무 길지 않은지)
2. 네거티브 프롬프트 확인
3. 해상도가 지원 범위인지 확인
4. ANLAS 잔액 확인
5. 로그 확인 (Options → Log Level)

#### 이미지가 원하는 대로 나오지 않을 때
1. Prompt Guidance 값 조절 (낮추거나 높이기)
2. Sampler 변경 시도
3. Steps 증가 (28 → 32~40)
4. 와일드카드 패턴 조정

#### Character Reference가 작동하지 않을 때
1. 참조 이미지가 올바른 해상도인지 확인
2. Fidelity 값 조절 (0.8 이상 권장)
3. Style Aware 설정 확인
4. Vibe Transfer와 동시 사용 중인지 확인 (동시 사용 불가)

### 파일명 포맷 설정

Options에서 생성 이미지의 파일명 형식을 커스터마이징할 수 있습니다.

**지원 플레이스홀더**:
- `[datetime]`: 날짜와 시간
- `[date]`: 날짜만
- `[time]`: 시간만
- `[prompt]`: 프롬프트 텍스트
- `[character]`: 캐릭터 프롬프트
- `[seed]`: 시드 값

**예시**:
- `[datetime]_[prompt]` (기본)
- `[character]_[date]_[seed]`
- `[prompt]_[datetime]`

프롬프트 입력 시 자동완성 기능을 사용할 수 있습니다.
